# 面试准备总策略

## 项目目标

准备充分后跳槽，目标是获得更高薪资、更长职业寿命和更强技术成长性的嵌入式/SoC 系统软件岗位。

## 当前主线程定位

本线程作为长期项目总控台，只负责：

- 岗位定位与阶段规划
- 简历和项目材料梳理
- 学习进度复盘
- 证据链整理
- 后期 JD 检索、岗位匹配和投递策略

不在本线程处理：

- 具体代码审查
- 具体面试题训练
- 大段工程实现细节
- 单个技术点长篇学习

上述内容放到独立子线程处理，子线程只把结论同步回本线程。

## 当前初始定位

暂定定位：

```text
SoC 底层系统软件工程师
方向：AI SoC / 高性能 SoC / 通信芯片系统软件
能力栈：ARM/RISC-V 裸机固件 + BSP/外设驱动 + PCIe/Linux + QEMU 验证
```

该定位是阶段性假设，不是最终结论。后续需要根据项目证据、简历材料、JD 和面试反馈持续修正。

## 当前证据基础

- SmartFusion2 MSS + MiV-RV32 双 ELF 固件项目，用户为主负责开发人。
- 日常工作覆盖裸机固件、Linux 驱动、硬件联调、协议、数据流、异常上报、重构等。
- Linux 方向以 PCIe 驱动和用户态控制层为主要证据，同时通过 SPI/UART/UIO 体现 BSP/外设覆盖面。
- QEMU 建模可作为差异化能力，不作为当前唯一主线。

## 阶段优先级

### 第一阶段：材料建档

目标：把事实整理清楚，不急着下岗位结论。

- 个人档案
- 项目边界
- 证据地图
- 风险和待确认问题

### 第二阶段：简历主线

目标：形成一版主简历和 2-3 个岗位定制版本。

- 主简历：SoC 底层系统软件
- 定向版本：Linux BSP/Driver
- 定向版本：MCU/SoC 固件
- 备用版本：QEMU/仿真验证

### 第三阶段：学习补强

目标：只补对岗位和简历证据有帮助的内容。

- SmartFusion2 主项目深挖材料
- Linux PCIe/BSP 驱动基础补强
- ARM/RISC-V/SoC 启动与体系结构
- QEMU 验证闭环沉淀

### 第四阶段：投递准备

目标：基于 JD 做匹配分析，再决定投递。

- 岗位检索
- JD 能力拆解
- 简历版本匹配
- 面试反馈复盘
